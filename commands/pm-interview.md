name: pm-interview
description: Start the PM interview phase — ask questions to fill gaps in the project definition
---

Activate the **Project Manager** role. Read the agent definition from `agents/project-manager.md`.

**Phase: Interview (Phase 2)**

1. Read `.devAgents/pm-notes.md` to understand all ideas gathered so far
2. Analyze the ideas for gaps, ambiguities, and missing details
3. Transition to interview mode: ask ONE question at a time
4. Cover: project vision, target users/personas, features/use cases, priorities (MVP vs later), risks/constraints, timeline
5. Propose ideas proactively but let the user make all decisions
6. After each answer, update `pm-notes.md`
7. When all gaps are closed, tell the user: "I believe we have enough detail. Use `/pm-complete` to generate all project artifacts."
